## Architecture Decisions

1. **Dashboard split into 6 files**: DashboardPage → data/ + components/ sub-structure.
   Chart components (SearchCharts, TrendCharts) are lazy-loaded via `next/dynamic({ ssr: false })`
   since recharts adds ~300KB to the bundle.

2. **AppError as the single error type**: All thrown values normalised to AppError via
   `normalizeError()`. The `isOperational` flag distinguishes expected errors (network, auth)
   from programming bugs, enabling different monitoring strategies.

3. **ErrorBoundary as class component**: React's error boundary API requires class components.
   The `onError` callback accepts `AppError` (not the raw Error), so logging is typed.

4. **next/navigation mock in test-setup.ts**: Prevents "useRouter called outside of Next.js context"
   errors in component tests. `next/image` and `next/link` mocks excluded (add per-test as needed —
   test-setup.ts must remain a `.ts` file, not `.tsx`, to avoid esbuild JSX issues).

5. **Coverage thresholds at 80%**: Applied to lines/branches/functions/statements.
   Stub pages (app/**/page.tsx) and layouts excluded — they have no logic to test.
